# Plan 03-01 — VTEX Integration Routes

## Goal

Implement `POST /api/vtex/hook` and `POST /api/vtex/feed/poll` — the two entry points that receive VTEX events and drive the full processing pipeline.

## Requirements

| ID | Description | How This Plan Satisfies It |
|----|-------------|---------------------------|
| HOOK-01 | `POST /api/vtex/hook` endpoint exists | `src/app/api/vtex/hook/route.ts` exports `POST` |
| HOOK-02 | Extracts orderId from multiple payload shapes | Calls `extractOrderId()` from `@/lib/hookParser` |
| HOOK-03 | Returns HTTP 200 after awaiting full pipeline | Pipeline is `await`ed synchronously before `return Response.json(...)` |
| HOOK-04 | Optional demo secret via `x-demo-hook-secret` header | Calls `isHookSecretValid(header, cfg)` from `@/lib/config` |
| HOOK-05 | Hook URL displayed in UI | Route path is `/api/vtex/hook`; UI constructs full URL in Phase 4 |
| FEED-01 | `POST /api/vtex/feed/poll` endpoint exists | `src/app/api/vtex/feed/poll/route.ts` exports `POST` |
| FEED-02 | Calls VTEX Feed API, processes each event | `vtexClient.getFeedItems()` + sequential `for...of` calling `processOrder()` |
| FEED-03 | Deduplicates feed events idempotently | `isDuplicate()` / `markProcessed()` from `@/lib/deduplicator` |
| FEED-04 | Commits feed handle after successful pipeline if auto-commit enabled | `vtexClient.commitFeedItems([item.handle])` after `processOrder()` when `cfg.autoCommitFeed && erpStatus !== 'ERROR'` |
| FEED-05 | Duplicates stored as `DUPLICATE_IGNORED` in event log | `upsertOrder({ erpStatus: 'DUPLICATE_IGNORED' })` + `appendEventLog(...)` |
| API-01 | Hook returns 200 with JSON | `Response.json({ received: true, orderId })` |
| API-02 | Feed poll returns summary | `Response.json({ processed, duplicates, errors, total })` |
| ERR-05 | Malformed hook payload returns 400 | `extractOrderId()` returns `undefined` → `Response.json({ error: '...' }, { status: 400 })` |

### Task 3-01-01 — Hook Route (`src/app/api/vtex/hook/route.ts`)

Create the POST handler for the VTEX hook endpoint. Must await the full pipeline before returning 200 (no fire-and-forget on Vercel — function is killed on response).

**Files:**
- `src/app/api/vtex/hook/route.ts` — create new

**Implementation steps:**

1. Create directory `src/app/api/vtex/hook/` and file `route.ts`.

2. Add runtime declaration and imports:
```typescript
export const runtime = 'nodejs';

import { getServerConfig, getMissingCredentials, isHookSecretValid } from '@/lib/config';
import { getConfigOverrides, upsertOrder, appendEventLog } from '@/lib/store';

import { extractOrderId } from '@/lib/hookParser';

import type { ErpOrderRecord } from '@/types';
```

3. Add `buildConfig()` helper:
```typescript
function buildConfig() {
  return { ...getServerConfig(), ...getConfigOverrides() };
}
```

4. Export `POST` handler. Parse JSON body, return 400 on malformed:
```typescript
export async function POST(request: Request): Promise<Response> {
  let body: unknown;
  try {
    body = await request.json();
  } catch {
    return Response.json({ error: 'Invalid JSON body' }, { status: 400 });
  }
```

5. Build config. Validate demo hook secret (HOOK-04). Return 403 if configured secret doesn't match:
```typescript
  const cfg = buildConfig();
  const secret = request.headers.get('x-demo-hook-secret');
  if (!isHookSecretValid(secret, cfg as Parameters<typeof isHookSecretValid>[1])) {
    return Response.json({ error: 'Forbidden' }, { status: 403 });
  }
```

6. Extract orderId (HOOK-02). Return 400 if payload not recognized (ERR-05). Log to event log:
```typescript
  const orderId = extractOrderId(body as Record<string, unknown>);
  if (!orderId) {
    appendEventLog({
      timestamp: new Date().toISOString(),
      source: 'HOOK',
      level: 'WARN',
      message: 'Hook payload received but orderId could not be extracted',
      payload: body,
    });
    return Response.json({ error: 'Cannot extract orderId from payload' }, { status: 400 });
  }
```

7. Seed the store record BEFORE calling processOrder. `processOrder` is a no-op if no record exists:
```typescript
  const now = new Date().toISOString();
  const record: ErpOrderRecord = {
    id: orderId,
    orderId,
    source: 'HOOK',
    erpStatus: 'RECEIVED',
    startHandlingStatus: 'NOT_STARTED',
    receivedAt: now,
    attempts: 0,
    timeline: [{
      timestamp: now,
      step: 'EVENT_RECEIVED',
      status: 'INFO',
      message: `Hook event received for orderId: ${orderId}`,
    }],
  };
  upsertOrder(record);
```

8. Log receipt to event log:
```typescript
  appendEventLog({
    timestamp: now,
    source: 'HOOK',
    level: 'INFO',
    message: `Hook received orderId: ${orderId}`,
    orderId,
  });
```

9. Check credentials AFTER seeding (record exists for error tracking). If missing, return 200 anyway — prevent VTEX retry loop on misconfigured endpoint:
```typescript
  const missing = getMissingCredentials(cfg as Parameters<typeof getMissingCredentials>[0]);
  if (missing.length > 0) {
    appendEventLog({
      timestamp: new Date().toISOString(),
      source: 'HOOK',
      level: 'ERROR',
      message: `VTEX credentials missing: ${missing.join(', ')}`,
      orderId,
    });
    return Response.json({ received: true, orderId, warning: 'credentials_missing' });
  }
```

10. Build vtexClient. Await pipeline synchronously (no fire-and-forget):
```typescript
  const vtexClient = createVtexClient(cfg as Parameters<typeof createVtexClient>[0]);
  await processOrder(orderId, 'HOOK', { vtexClient, config: cfg });

  return Response.json({ received: true, orderId });
}
```

- [ ] `curl -X POST http://localhost:3000/api/vtex/hook -H "Content-Type: application/json" -d '{"orderId":"test-001"}'` returns `{"received":true,"orderId":"test-001"}`
- [ ] Body `{}` (no orderId) returns HTTP 400 with `{"error":"Cannot extract orderId from payload"}`
- [ ] Non-JSON body returns HTTP 400
- [ ] If `DEMO_HOOK_SECRET=mysecret` in env and wrong/missing header, returns 403
- [ ] No `appToken` appears in terminal output
- [ ] `npx tsc --noEmit` passes

### Task 3-01-02 — Feed Poll Route (`src/app/api/vtex/feed/poll/route.ts`)

Create POST handler for manual feed polling. Key constraints: sequential item processing (no `Promise.all`), module-level lock flag with `try/finally`, commit only after successful pipeline.

**Files:**
- `src/app/api/vtex/feed/poll/route.ts` — create new

**Implementation steps:**

1. Create directories `src/app/api/vtex/feed/poll/` and file `route.ts`.

2. Runtime, lock flag, and imports:
```typescript
export const runtime = 'nodejs';

// Module-level lock prevents concurrent polls on a warm Vercel instance (PITFALL C5).
let pollInProgress = false;

import { getServerConfig, getMissingCredentials } from '@/lib/config';
import { getConfigOverrides, upsertOrder, getOrderByOrderId, appendEventLog } from '@/lib/store';
import { createVtexClient } from '@/lib/vtexClient';
import { processOrder } from '@/lib/orderProcessor';
import { isDuplicate, markProcessed } from '@/lib/deduplicator';
import { FEED_POLL_MAX_EVENTS } from '@/lib/constants';
import type { ErpOrderRecord } from '@/types';
```

3. Add `buildConfig()` helper (same pattern as hook route):
```typescript
function buildConfig() {
  return { ...getServerConfig(), ...getConfigOverrides() };
}
```

4. Export `POST` handler with 409 guard, wrapping the real work in `try/finally`:
```typescript
export async function POST(_request: Request): Promise<Response> {
  if (pollInProgress) {
    return Response.json(
      { error: 'Feed poll already in progress', code: 'POLL_LOCKED' },
      { status: 409 },
    );
  }
  pollInProgress = true;
  try {
    return await runPoll();
  } finally {
    pollInProgress = false;
  }
}
```

5. Implement `runPoll()` as a module-level async function:
```typescript
async function runPoll(): Promise<Response> {
```

6. Check credentials:
```typescript
  const cfg = buildConfig();
  const missing = getMissingCredentials(cfg as Parameters<typeof getMissingCredentials>[0]);
  if (missing.length > 0) {
    return Response.json({ error: 'VTEX credentials not configured', missing }, { status: 401 });
  }
```

7. Build vtexClient. Fetch feed items (catch network errors):
```typescript
  const vtexClient = createVtexClient(cfg as Parameters<typeof createVtexClient>[0]);
  let items;
  try {
    items = await vtexClient.getFeedItems(FEED_POLL_MAX_EVENTS);
  } catch (err) {
    const message = err instanceof Error ? err.message : String(err);
    appendEventLog({
      timestamp: new Date().toISOString(),
      source: 'FEED',
      level: 'ERROR',
      message: `getFeedItems failed: ${message}`,
    });
    return Response.json({ error: 'Failed to retrieve feed items', message }, { status: 502 });
  }

  if (items.length === 0) {
    return Response.json({ processed: 0, duplicates: 0, errors: 0, total: 0 });
  }
```

8. Sequential processing loop with dedup, seed, processOrder, and commit:
```typescript
  let processed = 0;
  let duplicates = 0;
  let errors = 0;

  for (const item of items.slice(0, FEED_POLL_MAX_EVENTS)) {
    const orderId = item.orderId;
    if (!orderId) continue;

    const dedupInput = {
      eventId: item.eventId ?? item.id,
      orderId,
      state: item.currentState ?? item.state,
      timestamp: item.currentChangeDate ?? item.date,
    };

    // Deduplication check (FEED-03)
    if (isDuplicate(dedupInput)) {
      duplicates++;
      // Store as DUPLICATE_IGNORED if not already in store (FEED-05)
      if (!getOrderByOrderId(orderId)) {
        const now = new Date().toISOString();
        const dupRecord: ErpOrderRecord = {
          id: `dup-${orderId}-${Date.now()}`,
          orderId,
          source: 'FEED',
          erpStatus: 'DUPLICATE_IGNORED',
          startHandlingStatus: 'NOT_STARTED',
          receivedAt: now,
          attempts: 0,
          timeline: [{
            timestamp: now,
            step: 'DUPLICATE_IGNORED',
            status: 'INFO',
            message: `Duplicate feed event ignored for orderId: ${orderId}`,
          }],
        };
        upsertOrder(dupRecord);
      }
      appendEventLog({
        timestamp: new Date().toISOString(),
        source: 'FEED',
        level: 'INFO',
        message: `Duplicate feed event ignored: orderId=${orderId}`,
        orderId,
      });
      continue;
    }

    // Mark processed before pipeline to prevent re-entry
    markProcessed(dedupInput);

    // Seed record BEFORE calling processOrder
    const itemNow = new Date().toISOString();
    const record: ErpOrderRecord = {
      id: orderId,
      orderId,
      source: 'FEED',
      erpStatus: 'RECEIVED',
      startHandlingStatus: 'NOT_STARTED',
      receivedAt: itemNow,
      attempts: 0,
      timeline: [{
        timestamp: itemNow,
        step: 'EVENT_RECEIVED',
        status: 'INFO',
        message: `Feed event received for orderId: ${orderId}`,
      }],
    };
    upsertOrder(record);

    try {
      await processOrder(orderId, 'FEED', { vtexClient, config: cfg });

      // Commit feed handle after successful pipeline (FEED-04)
      // Must re-fetch to get status written by processOrder
      const finalRecord = getOrderByOrderId(orderId);
      if (cfg.autoCommitFeed && item.handle && finalRecord && finalRecord.erpStatus !== 'ERROR') {
        try {
          await vtexClient.commitFeedItems([item.handle]);
        } catch {
          appendEventLog({
            timestamp: new Date().toISOString(),
            source: 'FEED',
            level: 'WARN',
            message: `Failed to commit feed handle for orderId: ${orderId}`,
            orderId,
          });
        }
      }
      processed++;
    } catch (err) {
      errors++;
      const message = err instanceof Error ? err.message : String(err);
      appendEventLog({
        timestamp: new Date().toISOString(),
        source: 'FEED',
        level: 'ERROR',
        message: `processOrder failed for orderId=${orderId}: ${message}`,
        orderId,
      });
    }
  }
```

9. Return summary (API-02):

}
