# Plan 03-02 — ERP Management Routes

## Goal

Implement all five ERP-facing routes: `POST /api/vtex/orders/[orderId]/start-handling`, `GET /api/erp/orders`, `GET /api/erp/orders/[orderId]`, `POST /api/erp/orders/[orderId]/reprocess`, and `POST /api/erp/orders/[orderId]/retry-start-handling`.

## Requirements

| ID | Description | How This Plan Satisfies It |
|----|-------------|---------------------------|
| API-03 | `POST /api/vtex/orders/[orderId]/start-handling` calls VTEX Start Handling | `vtexClient.startHandling(orderId)`, updates record + timeline |
| API-04 | `GET /api/erp/orders` returns filtered/searched list | `getAllOrders()` + in-memory filter/search/sort on query params |
| API-05 | `GET /api/erp/orders/[orderId]` returns single record | `getOrderByOrderId(orderId)`, 404 if missing |
| API-06 | `POST /api/erp/orders/[orderId]/reprocess` re-runs full pipeline | Reset status, append timeline entry, `await processOrder()` |
| API-07 | `POST /api/erp/orders/[orderId]/retry-start-handling` retries Start Handling | Guard `startHandlingStatus !== 'SUCCESS'`, call `vtexClient.startHandling()` directly |
| ERR-02 | VTEX 401/403 errors recorded in timeline and inbox | `VtexApiError.status` check; processOrder already surfaces via `GET_ORDER_ERROR` |
| ERR-03 | VTEX 404 recorded in timeline | Same VtexApiError path |
| ERR-04 | VTEX 429 recorded in timeline | Same VtexApiError path |
| ERR-06 | Errors appear in inbox status, timeline, event log | processOrder writes timeline; routes write event log on catch |
| SEC-04 | appToken never in server logs | No `console.log(cfg)` anywhere; only log safe fields |

5. Look up existing record. Return 404 if not found:

6. Check credentials:
```typescript

  const missing = getMissingCredentials(cfg as Parameters<typeof getMissingCredentials>[0]);
```

7. Build client and call Start Handling. On success update record and timeline:
```typescript
  const vtexClient = createVtexClient(cfg as Parameters<typeof createVtexClient>[0]);
  try {
    await vtexClient.startHandling(orderId);
    const r = getOrderByOrderId(orderId);
    if (r) {
      upsertOrder({ ...r, startHandlingStatus: 'SUCCESS' });
      setOrderStatus(r.id, 'START_HANDLING_SUCCESS');
      appendTimelineEntry(r.id, {
        timestamp: new Date().toISOString(),
        step: 'START_HANDLING_SUCCESS',
        status: 'SUCCESS',
        message: 'Manual Start Handling trigger succeeded',
      });
    }
    return Response.json({ ok: true, orderId, startHandlingStatus: 'SUCCESS' });
  } catch (err) {
    const message = err instanceof Error ? err.message : String(err);
    const r = getOrderByOrderId(orderId);
    if (r) {
      upsertOrder({ ...r, startHandlingStatus: 'ERROR', errorMessage: message });
      setOrderStatus(r.id, 'START_HANDLING_ERROR');
      appendTimelineEntry(r.id, {
        timestamp: new Date().toISOString(),
        step: 'START_HANDLING_ERROR',
        status: 'ERROR',
        message,
      });
    }
    appendEventLog({
      timestamp: new Date().toISOString(),
      source: 'SYSTEM',
      level: 'ERROR',
      message: `Start Handling failed for orderId=${orderId}: ${message}`,
      orderId,
    });
    return Response.json({ ok: false, orderId, error: message }, { status: 502 });
  }
}
```

**Acceptance criteria:**
- [ ] POST to non-existent orderId returns HTTP 404 with `{"error":"Order not found"}`
- [ ] POST with missing credentials returns HTTP 401
- [ ] Successful call returns `{"ok":true,"orderId":"...","startHandlingStatus":"SUCCESS"}`
- [ ] VTEX error returns HTTP 502 with error message
- [ ] `npx tsc --noEmit` passes

3. Export `GET` handler. Parse query params for filter/search/sort:
```typescript
export async function GET(request: Request): Promise<Response> {
  const { searchParams } = new URL(request.url);

  const source = searchParams.get('source');    // 'FEED' | 'HOOK' | null/'ALL'
  const status = searchParams.get('status');    // ErpStatus | null/'ALL'
  const search = searchParams.get('search');    // free text | null
  const sort   = searchParams.get('sort') ?? 'receivedAt_desc';

  let orders = getAllOrders(); // already sorted newest-first (INBOX-03)
```

4. Apply filters:
```typescript
  if (source && source !== 'ALL') {
    orders = orders.filter((o): boolean => o.source === (source as IntegrationSource));
  }
  if (status && status !== 'ALL') {
    orders = orders.filter((o): boolean => o.erpStatus === (status as ErpStatus));
  }
  if (search) {
    const q = search.toLowerCase();
    orders = orders.filter((o): boolean =>
      o.orderId.toLowerCase().includes(q) ||
      (o.sequence ?? '').toLowerCase().includes(q) ||
      (o.customerName ?? '').toLowerCase().includes(q) ||
      (o.erpPayload?.items ?? []).some(
        (item) => (item.name ?? '').toLowerCase().includes(q),
      ),
    );
  }
```

5. Apply sort (default is already newest-first from `getAllOrders()`):
```typescript
  if (sort === 'receivedAt_asc') {
    orders = [...orders].sort(
      (a, b) => new Date(a.receivedAt).getTime() - new Date(b.receivedAt).getTime(),
    );
  }
  // Other sort values use the default newest-first order

  return Response.json({ orders, total: orders.length });
}
```
